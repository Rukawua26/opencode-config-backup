# Working Agreement

- Keep skills global, but keep project specs local to each repo.
- Do not create `/home/miguel/spec/` as a global spec source.
- Skills should describe when to use them, the workflow, restrictions, and expected output.
- Avoid skills that load broad context automatically.
- Prefer small, composable skills over large all-purpose instructions.
- If a skill changes OpenCode behavior, document whether OpenCode must be restarted.
